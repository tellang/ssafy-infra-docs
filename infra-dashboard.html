계획된 인프라 항목들이 실제로 어떻게 구성되었는지 비교하고 검증합니다.

1

SSH 보안 설정 (Harden-SSH)

🤔 왜 필요한가요?

서버는 인터넷에 노출된 집의 '대문'과 같습니다. 비밀번호만으로 문을 잠그면, 해커들이 자동화된 프로그램으로 수백만 개의 비밀번호를 계속 시도해서(브루트포스 공격) 열려고 할 수 있습니다. 비밀번호 대신 암호화된 '열쇠'(SSH Key)만 사용하고, 가장 강력한 권한을 가진 '집주인'(root) 계정의 직접 로그인을 막으면 훨씬 안전합니다.

😱 설치/설정하지 않으면?

해커가 비밀번호를 추측하여 서버에 침투할 수 있습니다. 일단 들어오면 서버를 망가뜨리거나, 다른 곳을 공격하는 좀비 서버로 만들거나, 데이터를 훔쳐갈 수 있습니다.

🛠️ 어떻게 수동으로 설정하나요?

# 1. SSH 설정 파일을 엽니다.
sudo vi /etc/ssh/sshd_config

# 2. 아래 두 항목을 찾아 주석(#)을 제거하고 'no'로 변경합니다.
PasswordAuthentication no
PermitRootLogin no

# 3. 변경사항을 적용하기 위해 SSH 서비스를 재시작합니다.
sudo systemctl restart sshd
2

UFW 방화벽 설정

🤔 왜 필요한가요?

서버 안에는 여러 서비스(PostgreSQL, Redis 등)가 돌고 있습니다. 이 서비스들은 서버 내부의 다른 서비스들하고만 통신해야 합니다. 방화벽은 허락된 '문'(포트 22, 80, 443)을 제외한 모든 외부에서의 접근을 막는 '벽' 역할을 합니다.

😱 설치/설정하지 않으면?

누구나 우리 서버의 데이터베이스나 캐시에 직접 접근하여 데이터를 보거나 수정/삭제할 수 있게 됩니다. 심각한 데이터 유출 사고로 이어집니다.

🛠️ 어떻게 수동으로 설정하나요?

# 1. 기본적으로 들어오는 모든 외부 요청을 거부합니다.
sudo ufw default deny incoming

# 2. 우리가 꼭 열어줘야 하는 문(포트)만 허용합니다.
sudo ufw allow 22/tcp   # SSH 원격 접속용
sudo ufw allow 80/tcp   # HTTP 웹 트래픽용
sudo ufw allow 443/tcp  # HTTPS 보안 웹 트래픽용

# 3. 방화벽을 활성화합니다.
sudo ufw enable
3

Docker 설치

🤔 왜 필요한가요?

서비스를 각각 독립된 '컨테이너'라는 상자에 담아 실행합니다. 각 상자는 자신만의 프로그램과 라이브러리를 가지므로 서로 충돌하지 않고, 어디서든 동일하게 동작합니다.

🛠️ 어떻게 수동으로 설정하나요?

# 1. Docker 설치 스크립트를 다운로드하여 실행합니다.
curl -fsSL https://get.docker.com | sudo sh

# 2. 현재 사용자를 docker 그룹에 추가합니다.
sudo usermod -aG docker $USER
4

Nginx 설치 및 리버스 프록시

🤔 왜 필요한가요?

Nginx는 '교통정리 경찰'처럼, 사용자의 요청 주소(도메인)에 따라 적절한 내부 서비스(FE, BE 등)로 트래픽을 연결해주는 리버스 프록시 역할을 합니다.

🛠️ 어떻게 수동으로 설정하나요?

# 1. Nginx를 설치합니다.
sudo apt update && sudo apt install -y nginx

# 2. 리버스 프록시를 설정합니다 ( /etc/nginx/sites-available/default )
location / {
    proxy_pass http://127.0.0.1:7102;  # FE-main (선형 진입점)
}
location /dev/ {
    proxy_pass http://127.0.0.1:7202;  # FE-dev
}

# 3. 설정을 적용합니다.
sudo systemctl reload nginx
5

Jenkins 설치 (CI/CD)

🤔 왜 필요한가요?

코드 변경사항을 서버에 올리고 재시작하는 과정을 자동화하여, Push만 하면 자동으로 빌드와 배포가 이루어지게 해줍니다.

🛠️ 어떻게 수동으로 설정하나요?

# 1. Jenkins 데이터를 저장할 볼륨 생성 후 컨테이너로 실행합니다.
docker volume create jenkins_home
sudo docker run -d --name jenkins \
  -p 127.0.0.1:8080:8080 \
  -v jenkins_home:/var/jenkins_home \
  jenkins/jenkins:lts-jdk17
6

Tailscale VPN 설치

🤔 왜 필요한가요?

Jenkins처럼 외부에 포트를 열면 위험한 내부 관리 도구에 안전하게 접속하기 위해, 내 PC와 서버를 가상의 사설망으로 묶어줍니다.

🛠️ 어떻게 수동으로 설정하나요?

# 1. Tailscale 설치 및 로그인
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up

# 2. 할당된 내부 IP 확인
tailscale ip -4
