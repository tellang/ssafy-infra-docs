1. Network Architecture

외부 트래픽은 Nginx(80/443)를 통해 FE 컨테이너로만 진입합니다. 전체 서비스는 FE → BE → AI → External LLM 순으로 선형적인 요청 흐름을 가지며, 모든 서비스는 개별 Docker 컨테이너로 구성됩니다.

3. Data Isolation

Main과 Dev 환경은 완전히 분리된 Docker 환경(네트워크, 컨테이너)을 가집니다. BE는 PostgreSQL(읽기/쓰기) 및 Redis(세션/캐시)에, AI는 PostgreSQL(읽기)에 연결되며, FE는 DB/Redis에 직접 접근하지 않습니다.

5. Security Rules

Public Bind 제한

5432/6379/3306/27017/8080의 0.0.0.0 바인딩 금지.

SSH 강화

Password 로그인 비활성화. Root 직접 로그인 차단. Key-only 접근.

Jenkins 접근 제어

Nginx reverse proxy 경유. JENKINS_ALLOW_CIDR로 IP 제한.

Credential 관리

.env, .pem Git 추적 금지. Jenkins credential store 우선.

Tailscale Auth

Auth key 주기적 회전. Host-level VPN으로 운영자 접근 이중화.

Data Isolation

Main/Dev DB, Redis 완전 분리. Cross-environment access ACL 금지.
